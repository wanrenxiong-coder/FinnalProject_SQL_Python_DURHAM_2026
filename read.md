# Final Project Overview

## Project context
This repository is structured for a Master-level final project at Durham University with a clear separation between analysis code, datasets, and presentation materials.

## Current structure
- `CODE` — placeholder for analysis code, SQL queries, Python scripts, notebooks, or code modules.
- `DATA` — placeholder for dataset files, data extracts, cleaned data, and supporting resources.
- `PRESENTATION` — placeholder for the final presentation deck, report slides, and summary visuals.
- `read.md` — this documentation file for project explanation, methodology, and delivery instructions.
- `LICENSE` — the licence document for the repository.

> Note: At present, `CODE`, `DATA`, and `PRESENTATION` exist as placeholder files. For a complete project, these should be replaced by folders containing the actual assets.

## Recommended project contents
1. **CODE folder**
   - Python scripts for data processing, analysis, and modelling.
   - SQL scripts or notebooks demonstrating database queries and data extraction.
   - Reproducible analysis notebooks or code pipelines.

2. **DATA folder**
   - Raw dataset sources or references.
   - Cleaned datasets used for analysis.
   - Data dictionaries and schema documentation.

3. **PRESENTATION folder**
   - Final presentation slides.
   - Summary report or executive summary.
   - Visual outputs, charts, and dashboards.

## Academic presentation narrative
This project aims to demonstrate a complete data-driven research workflow:
- define the problem and research questions,
- collect and prepare the data,
- apply SQL and Python for analysis,
- interpret the results,
- present findings in a professional final report.
